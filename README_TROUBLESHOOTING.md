# 🔧 Устранение проблем с блокировщиком рекламы SufrShield

## Проблема: Блокировщик показывает низкий результат

Если ваш блокировщик рекламы показывает низкий результат на сайтах для тестирования, следуйте этим инструкциям:

### 🚨 Быстрая диагностика

1. **Откройте приложение SufrShield**
2. **Перейдите на экран "Блокировка рекламы"**
3. **Нажмите кнопку "Диагностика"**
4. **Проверьте результаты в консоли Xcode**

### 🔍 Пошаговое устранение проблем

#### Шаг 1: Проверка настроек Safari

1. Откройте **Настройки** на вашем iPhone/iPad
2. Перейдите в **Safari** → **Расширения**
3. Убедитесь, что **SufrShield Content Blocker** включен
4. Перезапустите Safari

#### Шаг 2: Обновление правил

1. В приложении SufrShield нажмите **"Обновить правила"**
2. Дождитесь завершения процесса
3. Проверьте, что количество правил увеличилось

#### Шаг 3: Проверка файлов

Убедитесь, что в проекте есть следующие файлы:

- ✅ `adblocker/adblock_rules.txt` - исходные правила AdBlock
- ✅ `adblocker/blockerList.json` - конвертированные правила
- ✅ `adblocker/ContentBlockerRequestHandler.swift` - обработчик расширения

#### Шаг 4: Тестирование

1. Откройте Safari
2. Перейдите на сайт с рекламой (например, youtube.com)
3. Проверьте, блокируются ли рекламные элементы
4. Используйте Safari Developer Tools для анализа

### 📱 Логи и отладка

#### Просмотр логов в Xcode:

1. Запустите приложение в Xcode
2. Откройте **Console** (View → Debug Area → Activate Console)
3. Ищите сообщения от **Content Blocker Extension**
4. Обратите внимание на сообщения с эмодзи (🚀, ✅, ❌, ⚠️)

#### Типичные сообщения:

```
🚀 Content Blocker Extension запущен
✅ Найдены правила в бандле (blockerList.json)
📊 Размер: 12345 символов
✅ JSON валиден, правил: 150
📤 Отправка правил из источника: Bundle
✅ Правила успешно отправлены
```

### 🛠️ Технические детали

#### Структура правил:

Правила должны иметь следующий формат:

```json
[
  {
    "trigger": {
      "url-filter": ".*ads.*",
      "resource-type": ["script", "image"]
    },
    "action": {
      "type": "block"
    }
  }
]
```

#### Поддерживаемые модификаторы:

- `$script` - блокировка скриптов
- `$image` - блокировка изображений
- `$stylesheet` - блокировка стилей
- `$third-party` - блокировка сторонних ресурсов
- `$domain=example.com` - ограничение по домену

### 🔧 Если проблема не решается

#### Переустановка:

1. Удалите приложение SufrShield
2. Перезагрузите устройство
3. Установите приложение заново
4. Включите расширение в Safari

#### Проверка правил:

1. Убедитесь, что правила не слишком сложные
2. Проверьте синтаксис AdBlock
3. Убедитесь, что нет конфликтующих правил

#### Альтернативные решения:

1. Используйте простые правила для тестирования
2. Проверьте работу с базовыми правилами
3. Убедитесь, что расширение загружается

### 📞 Поддержка

Если проблема не решается:

1. Соберите логи диагностики
2. Сделайте скриншот настроек Safari
3. Опишите, на каких сайтах проблема
4. Укажите версию iOS и Safari
